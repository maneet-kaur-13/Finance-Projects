# Insurance Risk Simulation & Solvency Analysis Using Monte Carlo Methods

This project simulates the annual loss distribution of an insurance portfolio and uses Monte Carlo methods to estimate tail-risk measures relevant to solvency analysis.

## Problem statement
Insurance companies collect premiums today but face uncertain future claims. Averages alone are not enough because claim losses are noisy and sometimes extreme. This project estimates:
- expected loss
- volatility of loss
- Value at Risk (VaR)
- Conditional VaR (CVaR)
- probability of underwriting loss
- probability of ruin given a capital buffer

It also evaluates stress scenarios where claim frequency and/or severity worsen.

## Core modeling choices
- **Claim frequency** is modeled using a **Poisson distribution**.
- **Claim severity** is modeled using a **Lognormal distribution** because losses are non-negative and right-skewed.
- **Portfolio loss** is generated by simulating many years and aggregating the claim amounts.
- **Solvency** is approximated using a capital buffer and checking how often losses push underwriting result below that buffer.

## Files
- `simulation.py`: simulation engine, risk metrics, stress testing
- `app.py`: command-line entry point
- `requirements.txt`: dependencies

## How to run
```bash
python app.py
```

Example with stress:
```bash
python app.py --freq_stress 1.25 --sev_stress 1.2 --scenario_name "Combined Stress"
```

## Example assumptions
- 1,000 policies
- annual claim frequency = 0.08 per policy
- average claim severity = 50,000
- severity volatility = 0.9
- premium per policy = 7,000
- fixed expenses = 1,200,000
- capital buffer = 2,500,000

## Limitations
- simplified one-period model
- no reinsurance layer
- no reserve dynamics
- no correlation across peril types
- parameter assumptions are illustrative
